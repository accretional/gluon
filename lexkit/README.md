# lexkit

lexkit is a self-hosting grammar toolkit that can parse any context-free
language from a static, serialized `GrammarDescriptor` in Protocol Buffers.

A `GrammarDescriptor` contains everything needed to parse a language: a
`LexDescriptor` (how to read the EBNF notation) and a set of
`ProductionDescriptor`s (the grammar rules). Because the grammar itself is
encoded in protobuf, parsing requires no hardcoded configuration — just
deserialize and call `Parse`.

## Self-hosting

The system is fully self-hosting. The EBNF grammar is written in EBNF,
parsed by its own `LexDescriptor`, and the resulting `GrammarDescriptor`
can re-parse the original source identically. This validates the full loop:

```
                       +-----------+
                       |  ebnf.txt |  (EBNF grammar written in EBNF)
                       +-----+-----+
                             |
                     Parse(src, EBNFLex())
                             |
                             v
                  +---------------------+
                  |  GrammarDescriptor  |  (protobuf: lex config + 13 productions)
                  +-----+-------+-------+
                        |       |       |
                        v       v       v
               .textproto  .binarypb  ToASCII()
               (46 KB)     (4.2 KB)   (1.2 KB)
                                        |
                                        v
                               reconstructed text
                                        |
                                Parse(text, lex)
                                        |
                                        v
                              identical GrammarDescriptor
```

All three LexDescriptors (EBNF, Go, Proto) are serialized as binary protobuf
and loaded via `//go:embed`. There is no hardcoded lexer configuration in Go.

## How it works

1. **`EBNFLex()` / `GoLex()` / `ProtoLex()`** — load a `LexDescriptor` from
   embedded `.binarypb` files via `//go:embed`.
2. **`Parse(src, lex)`** — reads EBNF source text and returns a
   `*pb.GrammarDescriptor` containing the lex config and all productions.
3. **`ToTextproto(gd)`** — serializes via `prototext.Marshal` (ASCII enum
   names like `VERTICAL_LINE` render automatically).
4. **`ToASCII(gd)`** — reconstructs EBNF source text from a
   `GrammarDescriptor`, using bit masking (`& 0x7F`) to convert ASCII enum
   values directly to bytes.
5. **`LoadGrammar(path)`** — reads a textproto back into a `GrammarDescriptor`.

The round-trip is validated for all three grammars:
`binarypb -> ToASCII -> Parse -> GrammarDescriptor` produces identical
productions to the original.

## Serialization formats

Each grammar is available in three formats, generated by `lexgen`:

| Format | EBNF | Go | Proto | Notes |
|--------|------|-----|-------|-------|
| `.txt` (source) | 1.2 KB | 9 KB | 3.8 KB | Human-authored EBNF |
| `.binarypb` | 4.2 KB | 28 KB | 12 KB | Full GrammarDescriptor |
| `.textproto` | 46 KB | 312 KB | 142 KB | Readable, with ASCII enum names |

The ~4x overhead in binarypb vs raw text comes from wrapping each character
in a `UTF8 { ascii: N }` message (4 bytes per char vs 1 byte raw). This is
the cost of character-level proto encoding. See [PROTOSCOPE.md](PROTOSCOPE.md)
for wire format analysis.

## Dev loop

```sh
# Parse all grammars and regenerate all output formats:
go run ./lexkit/cmd/lexgen ./lexkit

# Run tests (includes self-hosting and round-trip validation):
go test ./lexkit/...
```

## Proto types

All types live in `grammar.proto` and `lex.proto`, with character encoding
via `unicode/ascii.proto` and `unicode/utf_8.proto` (vendored from
[proto-symbol](https://github.com/accretional/proto-symbol)):

- **`GrammarDescriptor`** — a `LexDescriptor` + repeated `ProductionDescriptor`.
  This is a complete, self-contained representation of a context-free grammar.
- **`LexDescriptor`** — configures EBNF delimiters (definition, termination,
  alternation, brackets, quotes, comments) as `UTF8` values
- **`ProductionDescriptor`** — production name + `TokenDescriptor` body
- **`TokenDescriptor`** — repeated `UTF8` chars encoding the EBNF expression
- **`UTF8`** — oneof `{ASCII ascii, uint32 symbol}` — ASCII range uses
  readable enum names (e.g. `VERTICAL_LINE`, `EQUALS_SIGN`)
